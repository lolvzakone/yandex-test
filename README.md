Ответ на вопрос 7. Как можно доставить твой index.html в контейнер, не пересобирая образ?
Можно использовать команду docker cp для копирования файла в контейнер. Потребуется перезапустить только службу nginx внутри контейнера через команду docker exec.
Также можно использовать функцию монтирования раздела с командой mount , в данном случае дастоточно в смомнтированной директории изменить файл , и перезапустить службу nginx.

Пайплайн можно разделить на этапы:
1. Push в main ветку, на данном этапе Gitlab обнаруживает изменения в main ветке, отправляет webhook в Gitlab CI
2. Триггер : Предварительно написанный файл ci.yml описыввает этапы развёртывания, и запускает runnerы
3. Тестирование: Выполняются тесты, при неуспешном завершении теста, задача маркируется статсуом Failed, выполнение пайплайна останавливается, и отправляется уведомление в Telegram
4. Сборка образа:  При успешном выполнении тестов, Образ собирается  с тегом, который формируется из хэша коммита.
5. Пуш в Docker hub:  Отправка собранного образа в Docker Hub
